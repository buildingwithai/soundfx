how it works

Install it once. Run soundfx. Start assigning reactions.

Step 1

Install the command globally

The package installs once with npm so the soundfx command is available from your normal shell.

npm install -g @buildingwithai/soundfx

Step 2

Run soundfx

Open the CLI from the terminal itself. No extra dashboard required.

Step 3

Assign sounds to events

Map an event, test it, and keep adjusting until the shell feels right.

Built-in commands

  • soundfx list
  • soundfx sounds
  • soundfx assign <event> <sound>
  • soundfx test unknown_command

faq

Quick answers for the practical questions people ask before they install.

Is this a desktop app?

No. This package is the terminal product by itself. It installs as a CLI tool.

Can I hear a preview before I save?

Yes. You can test a mapped event and hear the result right away.

Does it work only in one shell?

No. It can work anywhere the same shell profile is loaded.

Can I change the sound later?

Yes. Reassign the event whenever you want and keep tuning the experience.

Do I need the desktop app?

No. The CLI tool stands on its own for terminal use.

Can I use my own sounds?

Yes. The terminal flow is designed around swapping and testing mappings.
